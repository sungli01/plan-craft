## 🔍 테스트 1: RAG 시스템 통합 확인

### 목표
Phase별 자동 자료 수집 및 참고 문헌 표시 확인

### 단계

#### 1.1 데모 모드 시작
```
1. 메인 페이지 접속
2. "🎮 데모 시작" 버튼 클릭
3. "사고과정 보기" 버튼 클릭
```

#### 1.2 RAG 로그 확인
**예상 로그 패턴**:
```
🔍 RAG 시스템 활성화: 요구사항 분석 관련 자료 수집 중...
✅ RAG: 5개 참고 자료 수집 완료
📄 참고1: [제목]
   ↳ [스니펫 80자]...
📄 참고2: [제목]
   ↳ [스니펫 80자]...
📄 참고3: [제목]
   ↳ [스니펫 80자]...
```

#### 1.3 Phase별 확인 체크리스트
- [ ] G1_REQUIREMENTS_ANALYSIS: "요구사항 정의 방법론 best practice" 검색 확인
- [ ] G2_DATA_COLLECTION: "시장 조사 데이터 분석 사례" 검색 확인
- [ ] G3_OUTLINE_CREATION: "보고서 구조 작성 가이드" 검색 확인
- [ ] 각 Phase마다 상위 3개 참고 자료 표시 확인
- [ ] 스니펫이 80자로 제한되어 표시되는지 확인

#### 1.4 예상 결과
✅ **성공**: 각 Phase 시작 시 RAG 시스템이 활성화되고 3개 참고 자료가 표시됨
❌ **실패**: RAG 로그가 보이지 않거나 참고 자료가 표시되지 않음

## 🤖 테스트 2: Agent 역할 명확화 확인

### 목표
Phase별 Agent의 전문 역할이 명확히 표시되는지 확인

### 단계

#### 2.1 Agent 역할 로그 확인
**예상 로그 패턴**:
```
👤 Master Orchestrator: 요구사항 분석 전략 수립 - 핵심 요구사항 식별 및 우선순위 결정
👤 Code Agent: 기술 요구사항 분석 - 시스템 아키텍처 초기 설계
👤 Quality Agent: 요구사항 검증 - 누락/모순 사항 점검 및 개선 제안
```

#### 2.2 Phase별 역할 변화 확인
**G1 (요구사항 분석)**:
- Master Orchestrator: 요구사항 분석 전략 수립
- Code Agent: 기술 요구사항 분석
- Quality Agent: 요구사항 검증

**G2 (자료 수집)**:
- Master Orchestrator: 데이터 수집 범위 정의
- Code Agent: 기술 스택 조사
- Quality Agent: 데이터 품질 검증

**G3 (개요 작성)**:
- Master Orchestrator: 보고서 구조 설계
- Code Agent: 기술 문서 구조 설계
- Quality Agent: 구조 논리성 검증

#### 2.3 체크리스트
- [ ] 각 Phase 시작 시 Agent 역할 로그 "👤" 표시 확인
- [ ] Master Orchestrator의 Phase별 다른 역할 확인
- [ ] Code Agent의 Phase별 다른 역할 확인
- [ ] Quality Agent의 "목표: 95%" 표시 확인
- [ ] DevOps Agent의 G8-G10에서의 역할 확인

#### 2.4 예상 결과
✅ **성공**: 각 Agent가 Phase에 맞는 전문 역할을 표시함
❌ **실패**: Agent 역할이 일관되게 같거나 표시되지 않음

## 📊 테스트 3: 사고과정 모니터링 강화 확인

### 목표
Agent 활동 + 구체적 작업 내용이 상세히 표시되는지 확인

### 단계

#### 3.1 상세 활동 로그 확인
**예상 로그 패턴**:
```
📝 요구사항 분석 [10%] [전략 수립 중] 사용자 요구사항 상세 분석 중
📝 요구사항 분석 [20%] [전략 수립 중] 핵심 기능 도출 및 우선순위 결정
📝 요구사항 분석 [30%] [방향성 정의] 제약사항 및 전제조건 파악
📝 요구사항 분석 [40%] [방향성 정의] 목표 지표(KPI) 설정
📝 요구사항 분석 [50%] [품질 기준 설정] 요구사항 검증 및 확정
```

#### 3.2 Agent별 활동 동사 확인
**Master Orchestrator**:
- 전략 수립 중
- 방향성 정의
- 품질 기준 설정
- 전체 조율
- 최종 승인

**Code Agent**:
- 아키텍처 설계
- 기술 검토 중
- 구현 계획
- 시스템 분석
- 기술 문서화

**Quality Agent**:
- 품질 검증 중
- 개선사항 도출
- 일관성 점검
- 완성도 평가
- 승인 준비

**DevOps Agent**:
- 배포 계획 수립
- 자동화 구성
- 인프라 점검
- 운영 준비
- 최종 배포

#### 3.3 10단계 상세 활동 확인
**G1 (요구사항 분석) 10단계**:
1. 사용자 요구사항 상세 분석 중
2. 핵심 기능 도출 및 우선순위 결정
3. 제약사항 및 전제조건 파악
4. 목표 지표(KPI) 설정
5. 요구사항 검증 및 확정
6. 기능 명세서 초안 작성
7. 이해관계자 검토 준비
8. 요구사항 추적 매트릭스 생성
9. 최종 요구사항 문서화
10. 요구사항 승인 완료

#### 3.4 체크리스트
- [ ] 로그 형식이 `[Agent 활동] 구체적 작업`으로 표시됨
- [ ] Agent별로 다른 활동 동사가 사용됨
- [ ] 각 Phase에 10개 스텝이 모두 표시됨
- [ ] 진행률 (10%, 20%, ... 100%)이 정확히 표시됨
- [ ] G1-G10 전체 Phase에서 상세 활동이 표시됨

#### 3.5 예상 결과
✅ **성공**: 각 스텝마다 Agent 활동 + 구체적 작업이 표시됨
❌ **실패**: 간단한 로그만 표시되거나 활동 동사가 없음

## 📄 테스트 4: 보고서 RAG 참고문헌 확인

### 목표
생성된 보고서의 부록 B.1에 RAG 수집 자료가 표시되는지 확인

### 단계

#### 4.1 프로젝트 완료 대기
```
1. 데모 프로젝트가 완료될 때까지 대기 (약 10-12분)
2. "프로젝트 완료!" 메시지 확인
3. "다운로드" 버튼 클릭
```

#### 4.2 다운로드 모달 확인
```
1. "HTML 다운로드" 또는 "PDF 다운로드" 선택
2. 파일 다운로드 완료 확인
3. 다운로드된 파일 열기
```

#### 4.3 부록 B.1 섹션 확인
**예상 내용**:
```html
부록 B. 참고 문헌 및 리소스

B.1 RAG 시스템 수집 자료
✓ 본 프로젝트를 위해 AI RAG 시스템이 자동으로 수집한 참고 자료입니다.

B.1.1 요구사항 분석 관련 자료
• [제목]
  [URL]
  [스니펫 설명]
  
• [제목]
  [URL]
  [스니펫 설명]

B.1.2 자료 수집 관련 자료
...

B.2 기술 문서
...
```

#### 4.4 체크리스트
- [ ] 부록 B.1 "RAG 시스템 수집 자료" 섹션 존재
- [ ] 안내 메시지 "본 프로젝트를 위해 AI RAG 시스템이..." 표시
- [ ] Phase별 참고 자료가 B.1.1, B.1.2, ... 로 구분됨
- [ ] 각 참고 자료에 제목, URL, 스니펫이 표시됨
- [ ] 최소 3-5개의 Phase별 자료 섹션이 있음
- [ ] 각 섹션에 최대 5개의 참고 자료가 표시됨

#### 4.5 예상 결과
✅ **성공**: 부록 B.1에 RAG 수집 자료가 상세히 표시됨
❌ **실패**: 부록 B.1이 없거나 RAG 자료가 표시되지 않음

## 🔧 문제 해결 가이드

### RAG 로그가 보이지 않는 경우
1. 브라우저 Console에서 `window.ragSystem` 확인
2. `window.ragSystem.searchWeb` 함수 존재 확인
3. 페이지 새로고침 후 재시도

### Agent 역할이 표시되지 않는 경우
1. "사고과정 보기" 버튼을 클릭했는지 확인
2. Terminal Console에서 "👤" 아이콘 검색
3. 로그가 너무 빨리 지나가는 경우 스크롤을 멈추고 확인

### 보고서에 RAG 자료가 없는 경우
1. 프로젝트가 완전히 완료되었는지 확인 (100%)
2. HTML 파일을 텍스트 에디터로 열어서 "B.1 RAG" 검색
3. PDF의 경우 북마크나 목차에서 "부록 B" 확인

### 일반적인 문제
- **페이지 로딩 느림**: 네트워크 상태 확인, 페이지 새로고침
- **버튼 클릭 안됨**: 페이지 완전 로딩 후 재시도
- **로그 너무 빠름**: 브라우저 개발자 도구에서 Console 로그 필터링 사용
